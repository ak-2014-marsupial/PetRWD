# FSD React + TypeScript Starter Kit

Цей проєкт є стартовим набором для розробки React-застосунків з використанням TypeScript та методології Feature-Sliced Design (FSD). Він створений за допомогою Vite.

**Ключові особливості архітектури:**
- **Модульність:** Завдяки FSD, нові фічі легко розробляти та інтегрувати.
- **UI Kit у shared:** Спільні, незалежні та перевикористовувані компоненти інтерфейсу (наприклад, `InputRange`, `Button`, `Input`) зібрані в `src/shared/ui/kit`. Це забезпечує єдиний вигляд та поведінку для базових елементів, мінімізує дублювання коду та спрощує перенесення фіч між проектами, оскільки `kit` виступає як єдиний пакет залежностей для UI.
- **Мінімум глобального стану:** На даному етапі не використовується глобальний state manager (як Redux або Zustand). Логіка, така як керування темою, реалізована переважно на рівні CSS, що спрощує розробку та підтримку модулів.
- **Потужна робота зі стилями (PostCSS та CSS-змінні):** Використання PostCSS дозволяє ефективно керувати CSS-змінними. Це не тільки забезпечує централізований контроль констант, пов'язаних зі стилями (наприклад, кольори, розміри шрифтів), але й надає значні переваги над JavaScript-підходом до темизації: висока продуктивність завдяки нативній обробці браузером, легкість підтримки, динамічна адаптація без повторного рендеру React-компонентів, та чисте розділення відповідальності між логікою (JS) та стилями (CSS). Це забезпечує узгодженість по всьому проєкту, зокрема завдяки файлу `appConfig.ts` для деяких ключових значень. *(Більш детальна інформація про систему кольорів знаходиться в `docs/colors.md`)*
- **CSS Modules:** Для ізольованого стилізування компонентів.
- **Динамічна маршрутизація:** Маршрути налаштовуються в одному місці (`app/config/routerConfig.tsx`) та автоматично використовуються для навігації.
- **Псевдоніми шляхів:** Налаштовані (`@/*` -> `src/*`) для зручних імпортів.

## Основні Функції

Цей стартовий набір включає кілька реалізованих фіч, які демонструють можливості архітектури. **Важливо відзначити, що всі ці фічі максимально використовують гнучкість та потужність CSS-змінних для динамічної адаптації інтерфейсу.**

### Керування Темою та Персоналізація
Застосунок надає два рівні керування темою:
- **Просте перемикання:** Фіча `ThemeSwitcher` дозволяє миттєво змінювати світлу/темну тему. *(Деталі реалізації в `docs/theme.md`)*.
- **Розширені налаштування:** Віджет `ThemeSettingsEditor` дозволяє користувачам обирати основний акцентний колір та глобальний розмір шрифту, забезпечуючи глибоку персоналізацію інтерфейсу. *(Деталі реалізації в `docs/ThemeSettingsEditor.md`)*.

Обидві фічі побудовані на основі CSS-змінних, що гарантує високу продуктивність. Вибір користувача зберігається в `localStorage`.

### Динамічна Навігація та Маршрутизація
Всі основні маршрути та елементи навігації визначені в `src/app/config/routerConfig.tsx`. Зміна цього файлу автоматично оновить як маршрути застосунку, так і навігацію бічної панелі.
*(Деталі про конфігурацію роутера знаходяться в `docs/routing.md`)*

### Модулі, Готові до Копіювання
`widgets` та `features` розроблені як високо незалежні. Наприклад, `widgets/app-shell` можна скопіювати до іншого проєкту, і з незначними корегуваннями його імпортів в `index.ts` він повинен функціонувати коректно.

## Структура Проєкту

```
src/
├── app/          # Загальні налаштування, стилі та провайдери
├── pages/        # Сторінки додатку
├── widgets/      # Складні, незалежні блоки (напр. AppShell)
├── features/     # Окремі бізнес-фічі (напр. ThemeSwitcher)
├── entities/     # Бізнес-сутності (напр. User, Product)
└── shared/       # Перевикористовуваний код (UI-кіт, утиліти, хуки, адаптери тощо)
```
*(Детальний опис кожного шару FSD знаходиться в `docs/fsd.md`)*

## Початок Роботи

1.  **Встановіть залежності:** `npm install`
2.  **Запустіть сервер розробки:** `npm run dev`

Застосунок буде доступний за адресою `http://localhost:5173`.
