# Система Кольорів та Рекомендації

Цей документ описує унікальну систему кольорів, що використовується в проєкті, та надає рекомендації щодо її правильного застосування.

## Філософія та Підхід

Система кольорів у цьому проєкті є **генеративною** та базується на **моделі HSL** (Hue, Saturation, Lightness). Замість того, щоб визначати статичний набір кольорів, ми визначаємо базові "настроювальні" параметри (відтінок, насиченість, яскравість), з яких потім процедурно генерується вся палітра.

Це дозволяє досягти неймовірної гнучкості. Наприклад, для зміни основного акцентного кольору всього додатку достатньо змінити лише одну змінну — відтінок (`--_h`).

## Концепція Шарів (Level) Глибини UI

Логіка номерів шарів проста: чим вищий номер шару (Level), тим темнішим і глибшим він стає у світлій темі, і навпаки — світлішим та виразнішим у темній темі. Ця інверсія відбувається автоматично, забезпечуючи правильний візуальний ефект при перемиканні тем.

## Структура Змінних для Кожного Шару

Для кожного з трьох шарів глибини доступний повний набір змінних, що дозволяє консистентно стилізувати будь-який компонент. Візьмемо Шар 2 як приклад (`X` = 2):

-   `--layer-X-bg`: Основний фон для цього шару.
-   `--layer-X-hover`: Фон при наведенні курсора.
-   `--layer-X-active`: Фон при активному стані (наприклад, натисканні).
-   `--layer-X-text`: Основний колір тексту на цьому фоні.
-   `--layer-X-text-secondary`: Вторинний, менш контрастний колір тексту.
-   `--layer-X-border`: Колір для рамок та розділювачів.
-   `--layer-X-accent`: Основний акцентний колір для інтерактивних елементів.
-   `--layer-X-on-accent`: Колір тексту, який слід використовувати *поверх* акцентного фону (наприклад, на кнопках).

## Рекомендації щодо Застосування

Завжди використовуйте змінні шарів. При стилізації нового компонента, **ніколи** не використовуйте жорстко закодовані кольори (hex, rgb). Замість цього, визначте, на якому шарі глибини знаходиться ваш компонент, і використовуйте відповідні змінні згідно з таблицею:

| Елемент / Контекст та Опис                                             | Background             | Text / Content         |
| :--------------------------------------------------------------------- | :--------------------- | :--------------------- |
| **Звичайний блок** <br> *Статичний вигляд картки або панелі.*           | `--layer-X-bg`         | `--layer-X-text`       |
| **Другорядні дані** <br> *Підписи, час, лічильники (менший контраст).* | `--layer-X-bg`         | `--layer-X-text-secondary` |
| **Елемент при наведенні** <br> *Реакція на курсор (м'яке фокусування).* | `--layer-X-hover`      | `--layer-X-text`       |
| **Елемент при кліку** <br> *Моментальний візуальний відгук на натискання.* | `--layer-X-active`     | `--layer-X-text`       |
| **Розділювач / Рамка** <br> *Лінії між повідомленнями або межі блоків.* | N/A                    | `--layer-X-border`     |
| **Активна кнопка / CTA** <br> *Головна дія на шарі (Надіслати, Видалити).* | `--layer-X-accent`     | `--layer-X-on-accent`  |
| **Тихий Акцент (текст/іконка)** <br> *Використання кольору акценту лише для тексту/іконки.* | `--layer-X-bg`         | `--layer-X-accent`     |

**Важливо:** Думайте "шарами", а не "кольорами". Ваше завдання як розробника — не підбирати кольори, а правильно визначити місце компонента в ієрархії глибини. Система сама подбає про правильні кольори для світлої та темної теми.
