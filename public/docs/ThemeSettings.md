## Динамічна Адаптація Кольорів

### 1. Призначення фічі: Дизайн-інструмент (Dev-only)

Цей компонент не є частиною користувацького функціонала для кінцевого споживача. Це **інструмент для розробки та комунікації з замовником**:

* **Уточнення кольорів:** Фіча дозволяє "погратися" з відтінками прямо в браузері разом із замовником, щоб миттєво знайти ідеальний візуальний баланс, не змінюючи код щоразу.
* **Тимчасова надбудова:** Компонент працює як динамічний шар поверх статичних значень, що визначені у файлі `src/styles/variables.css`. Він просто перекриває їх значеннями з `style` атрибута тега `<html>` або `<body>`.
* **Перенесення в код:** Як тільки ідеальні параметри знайдено (наприклад, h=210, s=50\% ), ці значення просто копіюються та прописуються як дефолтні у файл `src/styles/variables.css`.
* **Видалення з продакшену:** Після фіналізації дизайну цей компонент (разом із сервісом та обробниками) можна повністю видалити з кодової бази або виключити з фінального бандлу, оскільки його роль чисто конфігураційна на етапі прототипування.

### 2. Пряма взаємодія з DOM через CSS-змінні

Основний механізм роботи — це динамічна маніпуляція **CSS Custom Properties** (змінними) у реальному часі. Замість зміни класів чи інлайнових стилів для кожного елемента окремо, код через сервіс `themeVarsService` записує значення безпосередньо в об'єкт `style` кореневого елемента (наприклад, `:root`).

Код керує наступним переліком базових змінних, на яких будується вся палітра:

H (Hue): --h — базовий тон (від 0 до 360 градусів на колірному колі).

S (Saturation): --s, --acc-s, --text-s — насиченість (інтенсивність кольору від 0% сірого до 100% чистого кольору).

L (Lightness): --l-base, --acc-l, --text-l — яскравість/освітленість (від 0% чорного до 100% білого).

* `--h` — базовий тон (Hue)
* `--s` — загальна насиченість (Saturation)
* `--l-base` — базова яскравість (Lightness)
* `--l-step` — крок зміни яскравості для шарів
* `--acc-s` та `--acc-l` — параметри акцентних кольорів
* `--text-s` та `--text-l` — параметри основних текстових блоків
* `--text-s-step` та `--text-l-step` — кроки модифікації тексту

Коли ви рухаєте повзунок, React викликає `themeVarsService.set()`, який миттєво оновлює ці змінні в DOM. Весь інтерфейс, що використовує ці змінні у своїх `.css` файлах, синхронно змінює колір без перевантаження сторінки.

### 3. Збереження пресетів

Для зручності ітерацій реалізовано механізм `localStorage`. Це дозволяє розробнику або дизайнеру оновити сторінку або повернутися до роботи наступного дня, не втративши "накручені" раніше параметри, оскільки вони автоматично підтягуються з JSON-об'єкта в браузері.
