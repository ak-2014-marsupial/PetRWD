
# PetRWD

**Легкий React + TS стартер для пет-проєктів: адаптивність, HSL-теми та FSD-архітектура.**

**PetRWD** — це інструмент для розробників, які цінують швидкий старт

---

### Ключові особливості

* **Zero State Management:** Жодного Redux, Zustand чи MobX. Логіка тем та адаптивності працює нативно через **CSS Variables** та **PostCSS**.
* **HSL Color Engine:** Вбудований редактор кольорів, що базується на математиці HSL. Змінюйте палітру в реальному часі без перерендеру компонентів.
* **Справжня модульність (FSD-lite):** Кожна фіча — це ізольований модуль. Потрібно перенести функціонал в інший проєкт? Просто скопіюйте папку фічі та `shared/ui/kit`.
* **Responsive by Default:** Адаптивна сітка та гнучкі стилі, налаштовані для швидкої розробки пет-проєктів.

---

### Чому CSS-логіка краща за JS-стейт?

У **PetRWD** ми довіряємо управління візуальним станом браузеру. Ось чому це вигідно:

1. **Продуктивність (Zero Re-renders):** Зміна теми через React Context змушує перемальовувати все дерево компонентів. У нашому випадку змінюється лише значення CSS-змінної у `:root` — браузер робить це миттєво на рівні графічного рушія.
2. **Гнучкість HSL:** Замість зберігання сотень HEX-кодів, ми керуємо лише компонентами кольору (). Одна зміна `--_h` — і весь інтерфейс гармонійно змінює тон.
3. **Чистий UI-Kit:** Компонентам не потрібно знати про поточну тему через пропси чи хуки. Вони просто використовують `var(--layer-1-bg-color)`. Це робить їх максимально придатними для копіпасту між проєктами.
4. **Легкість підтримки:** Всі константи (кольори, брейкпоінти) зберігаються в одному файлі `.ts`, що дає типізацію під час розробки, але перетворюється на чистий CSS у браузері.

###  Архітектура (Modular Copy-Paste)

Проєкт наслідує принципи **Feature-Sliced Design**, де кожна фіча самодостатня:

```text
src/
  ├── app/              # Ініціалізація, глобальні стилі, роутинг
  ├── pages/            # Компоненти сторінок
  ├── features/         # Модулі (напр. ThemeEditor) — точка входу index.ts
  └── shared/
      ├── ui/           # кнопки, інпути ...
      └── constants/    # TS-файл з константами для PostCSS та RWD

```

### Роутинг (HashRouter)

Для забезпечення коректної роботи на статичних хостингах (наприклад, **GitHub Pages**) у проєкті використовується `HashRouter`.

**Чому HashRouter?**
1. **Сумісність з GitHub Pages:** Статичні сервери не знають, як обробляти прямі посилання React (наприклад, `/docs/about`). Використання хешу (`#/docs/about`) дозволяє уникнути помилки 404 при оновленні сторінки, оскільки браузер не надсилає частину після `#` на сервер.
2. **Zero Config:** Не потребує спеціального налаштування `_redirects` або конфігурації веб-сервера.
3. **Автономність:** Маршрутизація повністю керується клієнтським кодом, що ідеально підходить для архітектури PetRWD.
